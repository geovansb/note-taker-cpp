# Architecture Review — 2026-04-21

Findings from a full codebase review. Each finding is decomposed into
actionable tasks ordered by dependency and impact.

---

## F1 — String-based status protocol

Status communication between AppController and AppDelegate uses magic emoji
strings. Both sides parse the same prefixes. A typo silently breaks the UI.

### Tasks

- [ ] **F1.1** Define `enum class AppStatus` in a new `src/app_status.h` with
      all states (Idle, Loading, Dictating, Transcribing, Recording,
      Capturing, Listening, Finalizing, Beep, Error) and a
      `std::string statusLabel(AppStatus)` formatter.
- [ ] **F1.2** Change `AppController::setOnStatusChange` to emit `AppStatus`
      (+ optional error string) instead of `std::string`.
- [ ] **F1.3** Update `AppDelegate` to switch on the enum instead of
      `hasPrefix:` string matching.
- [ ] **F1.4** Remove all magic status strings from `app_controller.cpp`.

---

## F2 — Data races on VAD/assembler parameters

`Vad::threshold_`, `Vad::gain_`, and `ChunkAssembler::silence_timeout_s_`
are written from the main thread and read from the audio thread without
synchronization. Formally UB in C++17.

### Tasks

- [ ] **F2.1** Change `Vad::threshold_` and `Vad::gain_` to
      `std::atomic<float>`. Update setters/getters.
- [ ] **F2.2** Change `ChunkAssembler::silence_timeout_s_` to
      `std::atomic<float>`. Update setter/getter.
- [ ] **F2.3** Verify no other non-atomic cross-thread field access remains.

---

## F3 — WhisperWorker leaks whisper.cpp dependency in header

`whisper_worker.h` declares `struct whisper_context* ctx_` directly,
forcing every includer to see the whisper.cpp type. Other modules
(AudioCapture, AppController) use PIMPL correctly.

### Tasks

- [ ] **F3.1** Move all private fields of `WhisperWorker` into a
      `struct WhisperWorker::Impl` defined only in `whisper_worker.cpp`.
      Header keeps only `Impl* impl_` and the public API.

---

## F4 — WhisperWorker untestable (requires real model)

The bounded queue, drop-oldest policy, dictation vs. session dispatch, and
`on_session_done_` drain mechanism are pure concurrency logic locked behind
`whisper_init_from_file`. No unit test can exercise them.

### Tasks

- [ ] **F4.1** Extract a `TranscribeFunc` callback type
      (`std::function<int(const float*, int, ...)>`) that WhisperWorker
      calls instead of `whisper_full()` directly. Production injects the
      real whisper call; tests inject a fake.
- [ ] **F4.2** Add `tests/whisper_worker_test.cpp` covering: enqueue +
      dequeue ordering, drop-oldest on overflow, dictation callback fires,
      session-done callback fires when queue drains, stop() drains before
      joining.

---

## F5 — AppController is a God Object / untestable state machine

The mode state machine (IDLE/DICTATING/TRANSCRIBING/RECORDING/FINALIZING)
is the most complex logic in the codebase but can't be tested without a
real whisper model, mic, and filesystem.

### Tasks

- [ ] **F5.1** Extract `enum class Mode` and the state machine transitions
      into a standalone `src/mode_machine.h/.cpp` with a pure API:
      `tryDictate()`, `finishDictation()`, `tryRecord()`, `stopRecord()`,
      `finalize()`, `reset()`. Each returns bool (transition accepted).
      Emits `AppStatus` via callback on transitions.
- [ ] **F5.2** Refactor `AppController` to delegate mode decisions to
      `ModeMachine` instead of inline `compare_exchange_strong` calls.
- [ ] **F5.3** Add `tests/mode_machine_test.cpp` covering all valid
      transitions and all rejected transitions (e.g. dictate while
      recording, record while dictating, double-start, double-stop).

---

## F6 — OutputWriter: O(n^2) I/O + no tests

`flush()` rewrites the entire JSON+TXT from scratch on every chunk.
Also, segment accumulation logic has no test coverage.

### Tasks

- [ ] **F6.1** Add `tests/output_writer_test.cpp` using a temp directory:
      addSegment, flush, verify JSON content, verify TXT content, multiple
      flushes accumulate correctly.
- [ ] **F6.2** Add `tests/wav_writer_test.cpp`: write a known buffer, read
      back header + samples, verify correctness.
- [ ] **F6.3** (Optional, low priority) Switch JSON output to append-only
      JSONL to avoid O(n^2) rewrites. Requires updating the reader in M7.

---

## F7 — No audio recovery after config change failure

If `installTap()` fails after a device change, the app is left without
audio capture and no retry is attempted.

### Tasks

- [ ] **F7.1** Add a retry loop (up to 3 attempts with backoff) in the
      config-change notification handler when `installTap()` fails.
- [ ] **F7.2** If all retries fail, emit a persistent error status so the
      user knows audio is dead and a restart is needed.

---

## F8 — Raw OutputWriter* ownership semantics

WhisperWorker holds a raw pointer to OutputWriter whose lifetime is managed
externally by AppController. The API doesn't communicate this contract.

### Tasks

- [ ] **F8.1** Replace `OutputWriter*` with `std::shared_ptr<OutputWriter>`
      in WhisperWorker. AppController keeps its own `shared_ptr`. Eliminates
      the implicit lifetime coupling.
